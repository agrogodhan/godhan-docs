🔵 BLE Provisioning — Step by Step
1
New collar powered ON for first time
ESP32-S3 boots into BLE Provisioning Mode. It starts advertising its BLE name like "CattleCollar-045". WiFi is OFF at this point to save power.
BLE ADVERTISING ONE TIME
2
Farmer opens app → Scans for collars
App scans BLE and shows a list of unconfigured collars nearby. Farmer selects CattleCollar-045 and assigns it to a specific cow (Cow ID, name, age).
BLE SCAN GATT CONNECT
3
App sends WiFi credentials to collar via BLE
App writes WiFi SSID + Password to the collar's BLE GATT characteristic. Data is encrypted. Collar saves credentials to its internal flash memory (NVS storage).
GATT WRITE CREDENTIALS SAVED
4
Collar disconnects BLE · Connects to WiFi
BLE is turned OFF permanently (until factory reset). Collar uses saved credentials to join the farm WiFi network and gets an IP address from the router.
WiFi JOIN IP ASSIGNED
5
Collar registers itself on cloud server
First WiFi packet sent to cloud: collar sends its Device ID + Cow ID + firmware version. Cloud registers the device. From now on all data flows directly collar → cloud.
DEVICE REGISTER MQTT CONNECT
✓
Setup complete — BLE never used again
Every power cycle, collar auto-connects to WiFi using saved credentials. No phone needed nearby. Data flows directly to cloud every 30 seconds forever. BLE only re-activates if farmer presses factory reset button.
PERMANENT AUTO RECONNECT DIRECT TO CLOUD

// POST to: https://api.cattlecloud.com/v1/data · Every 30 seconds

{
  "device_id": "COLLAR-045", // unique collar ID flashed at factory
  "cow_id": "COW-112", // assigned during BLE setup
  "timestamp": "2026-04-12T08:32:10Z",

  // ── ACTIVITY ──────────────────────────────
  "steps": 142, // since last packet
  "posture": "grazing", // standing | lying | grazing | walking
  "activity_score": 74, // 0–100

  // ── HEALTH ────────────────────────────────
  "body_temp_c": 38.6, // normal: 38.0–39.5°C
  "heart_rate_bpm": 68, // normal: 48–84 bpm
  "spo2_pct": 97, // blood oxygen %
  "rumination_cycles_hr": 420, // jaw chews per hour

  // ── REPRODUCTION ──────────────────────────
  "vaginal_temp_c": 38.9, // rises during estrus
  "estrus_flag": false, // true = heat detected
  "calving_alert": false, // true = imminent calving

  // ── DEVICE STATUS ─────────────────────────
  "battery_pct": 82,
  "wifi_rssi": -58, // signal strength · used for zone detection
  "zone": "shed" // shed | open · derived from RSSI of APs
}
